# 프로젝트 회고 — 무엇을 했고, 무엇이 통했나

작성 2026-07-13 (모델 동결 시점). 수치 출처: `PROJECT_MEMORY.md`,
`phase2_results.md`, `results/phase3/*.md` (충돌 시 FROZEN_MODEL.md 우선).

## 한 문단 요약

Kang et al. 2025(MLB 투수 TJS 예측, AUC 0.93)를 완전 재현하는 데서
시작해, 그 수치가 배치 상황에서는 성립하지 않는 회고적 아티팩트임을
해체로 입증했고, 처음부터 전향(prospective) 문제로 다시 설계해 정직한
baseline 0.64를 얻은 뒤, 체계적인 개선 캠페인과 두 차례 외부 감사를
거쳐 **조건부 backtest ROC ~0.70 (경계 민감도 포함 실질 대역
0.60-0.70)의 동결 모델 + 경보 정책**으로 마무리했다. 4개월치 작업에서
시도한 수십 개의 아이디어 중 살아남은 것은 소수였고, 그 소수가 무엇인지
정확히 아는 것이 이 프로젝트의 실제 산출물이다.

## 숫자의 여정 — 왜 내려갔고, 왜 그게 전진이었나

| 단계 | 대표 수치 | 의미 |
|---|---|---|
| Kang 논문 | AUC 0.93 / R² 0.783 | 회고 설계의 수치 (액면 그대로는 배치 성능 아님) |
| 우리 재현 | AUC 0.92 / R² 0.783 | 재현 성공 — 이후 모든 해체의 신뢰 기반 |
| 해체 | 분류 ~0.11이 이력길이 아티팩트, 회귀 = 개인 내 보간 | 0.93의 출처 규명 — "잘못"이 아니라 "다른 질문"이었음 |
| 정직 회고 | ~0.60 (저자 v9 0.816 → 라이브 라벨 재추출 0.601) | 라벨 성숙까지 교정하면 회고도 이 수준 |
| 전향 재설계 | 0.64 (workload LR, 미래 시즌 test) | 배치 가능한 질문으로 바꾼 최초의 정직 수치 |
| 개선 캠페인 후 | **0.701 / 0.696 (H90/H150, 조건부 backtest)** | M_sa + hazard + 2024 확장; safety 경계에서 0.660/0.665 |
| 최종 표기 | sensitivity envelope ~0.60-0.70 (공식 CI 아님) | 경계·적합 불확실성 민감도를 합친 휴리스틱 대역; novel 투수 0.56-0.66은 KBO 예측이 아니라 MLB novel stress 참조 |

수치가 내려간 단계는 전부 "교정이 낮춘" 것이고(라벨 성숙, 누수 제거,
전향 전환), 올라간 단계는 전부 paired 검정과 rolling 방향 일관성을
통과한 것이다. 이 구분이 가능했다는 것 자체가 프로토콜의 성과다.

## 효과가 있었던 것

**1. 재현을 먼저 끝까지 한 것.** Kang을 0.92까지 재현했기 때문에
"우리가 못 만든 게 아니라 설계가 다른 것"이라고 말할 자격이 생겼다.
해체 결과(이력길이 아티팩트, 개인 내 보간)도 재현 코드 위에서만
증명할 수 있었다. 이 baseline은 체크포인트로만 쓰고 인용하지 않는다.

**2. 질문을 전향으로 다시 세운 것.** 월별 결정일(4-9월 1일), 경력
≥20경기 + 최근 365일 등판, 라벨 = (t, t+H] 내 TJS, 미래 시즌 test.
이 재설계가 이 프로젝트의 가장 큰 단일 기여다 — 이후의 모든 수치가
"구단이 실제로 쓸 수 있는 형태"에 대한 것이 되었다.

**3. 단순한 사용량·이력 feature + 로지스틱 회귀.** 끝까지 살아남은
유일한 feature 계층: 만성 사용량(pc_chronic), 급성 이탈(pc_acute_dev),
휴식(days_since_last), 구속 추세(vel_trend), 역할(start_share), 직전
시즌 노출(prior_pc_rate), 경력 길이(ncg_log). 특히 **M_sa(이력 스칼라
3종 추가)는 프로젝트에서 유일하게 유의한 additive 개선**(+0.017 EXCL0,
rolling 6/6 fold 방향 일치)이었다. 해석: "현재 사용량은 낮은데 직전
시즌 노출이 높았던 투수"가 위험 — 감소 패턴의 조기 관측.

**4. hazard supermodel — 성능이 아니라 구조 때문에 채택.** ROC 이득은
미미했지만 P(90일) ≤ P(150일) 정합을 구조적으로 보장한다(이진 쌍은
test 창의 47%에서 위반). 배치 지표라면 이 정합은 성능만큼 중요하다.

**5. 동결 프로토콜과 사전 등록.** 투수-clustered bootstrap(seed 0)을
전 변형이 공유하는 paired 검정, 새 변형 전 anchor 게이트 재현, 채택
기준(CI 0 배제 또는 rolling 방향 일관성) 사전 고정. 수십 개 변형을
시험하면서 거짓 양성에 속지 않은 것은 이 장치 덕분이다. 진단 프로브가
유망해 보였던 M_bf가 본검정에서 죽었을 때, 프로토콜이 없었다면
채택했을 것이다.

**6. 외부 감사를 받고, 감사도 검증한 것.** codex 감사 3회가 잡아낸
것: 2025 라벨 미성숙 포함(대표 수치 철회로 이어진 실수), "이중계상
제거"·"보정 확보"·"2-3배" 과장 표현, 선택 낙관 미측정, 2025의
전향성 상실, quota의 safety 경계 게이트 실패, A-IL parser의 elbow
소급(33건), 2026 사후 채점을 "진짜 전향"이라 부른 과장. 전부
수용했지만 **수치 주장은 전부 재계산으로 확인 후** 수용했다.
감사-검증-수용 루프가 문서의 신뢰도를 만들었다.

**7. "구단이 이미 안다" 반론에 실측으로 답한 것 (B'·blackout).**
H150 경보의 60%가 사용량 정상 시점에 나왔고(lead 중앙값 72일), 급성
감소 flag 단독은 무판별 — 즉 모델은 "이미 아픈 투수 리스트"의 재포장이
아니다. 같은 장치(60일 blackout)가 A-IL의 표면적 대박(+0.068)을
"공개 진단의 메아리"로 정확히 판정했다. **이 두 결과가 조기경보
주장의 실증 기반이다.**

**8. 불펜 문제를 배분으로 다시 정의한 것.** "불펜은 구조적으로
불가"라는 초기 결론은 틀렸다 — RP-내 판별력은 있었고(ROC ~0.65),
문제는 top-50이 전부 선발에게 가는 배분이었다. 이 재정의 자체는
유효했다. 다만 구체적 quota(RP 예약 20석)는 primary 경계에서만 Pareto
개선(RP 0→5, 총 손실 2)이고 **safety 경계 H150에서 게이트 실패(손실
4)** — 3차 감사 후 canonical에서 challenger로 강등했다. 남은 교훈:
정책 층의 해법도 모델과 같은 수준의 경계 민감도 검증을 받아야 한다.

**9. 2024 데이터 확장.** 사건 52→75/80으로 늘려 추정 정밀도를 올렸고,
M_sa 채택의 근거를 연도 부분집합 전체에서 확인할 수 있게 했다. 단,
0.644→0.677의 외형 상승은 test 구성 변화이지 모델 개선이 아니다 —
이 구분을 기록해 둔 것도 유효했다.

## 효과가 없었던 것

**1. 트래킹(Statcast) feature 전 계층.** release position, extension,
spin — 달력창·경기창·관측 부분집합 3중 확인에서 일관된 증분 없음.
tree 모델과 결합해도, 콘텐츠-only로도(~0.5) 마찬가지. **"고급 트래킹
데이터가 있어야 예측이 된다"는 서사는 우리 스케일에서 기각**됐고,
KBO 협상 논거도 "트래킹의 증분" → "내부 라벨·실제 역할·숨은 투구량과
결합한 검증 계약"으로 바꿨다.

**2. 복잡한 모델 전부.** HistGBM(동급-악화, 그리드 확장 무효), 시퀀스
딥러닝(사건 규모 부족), full GAM, 역할 상호작용/역할별 분리/stitch.
사건 ~200개 스케일에서 모델 복잡도는 전부 비용이었다.

**3. 문헌에서 빌린 아이디어 대부분.** Mayo식 추세(유의 악화),
Mastroianni식 변동성(널), pitch-mix/slider 서사(유의 악화), Cohen
release-side drift(전향 널), rest 구조(악화), torque 가중(pc와
r=0.9998로 무효), 경기 내 구속 감쇠(널). **회고 case-control에서 나온
신호는 전향 배치 프레임으로 거의 이전되지 않았다** — 이것이 이
프로젝트에서 가장 반복적으로 확인된 패턴이고, 레퍼런스 검증 원칙
(setup 대조 후 채택, 수치 이식 금지)이 필요한 이유였다.

**4. 의료 이력 지름길.** 팔꿈치-IL 이력은 +0.068로 보였지만 blackout
60일에서 음수로 붕괴 — 마지막 IL→수술 중앙값 34일, 즉 이미 공개된
진단의 반영이었다. 조기경보로는 무가치하고 triage(공개 진단 후 순위화)
로만 유효. prior_tjs는 방향은 양이지만 CI가 0을 포함 — 열린 후보로
남김(사건 축적/KBO에서 1순위 재검정).

**5. 2025 데이터를 서둘러 넣은 것 (실수).** 시트가 2026-07까지
이어진다는 사실을 수집 완결성으로 착각해 자체 감사(라벨 신뢰 종료
2024-12-31)를 위반했다. 외부 감사가 잡았고, 대표 수치를 철회·정정했다.
교훈은 단순하다: **자기 감사 결과는 자기에게도 적용해야 한다.**

**6. 단일 수치 인용.** 0.70 하나로 말하는 순간 경계 선택(safety에서
0.66), 연도 구성(2024 강세 0.80 vs 2023 0.62), novel 투수(0.66→0.56)에
대한 민감도가 전부 숨는다. 최종 표기를 "조건부 backtest + 대역"으로
바꾼 것이 감사 이후 가장 중요한 서술 변화다.

## 최종 산출물 (동결 기준)

- **모델**: M_sa 9-feature + discrete-time hazard, 계수·재보정 고정 —
  `results/phase3/FROZEN_MODEL.md` + `frozen_model_state.json` (정본).
- **경보 정책**: 월별 top-50 (q=0 canonical, 순위 = P150). RP 예약
  20석은 safety 경계 실패로 challenger로만 보존.
- **검증 아카이브**: 2025 점수 timestamp (`frozen_scores_2025_*.csv`,
  해시 고정) — 라벨 성숙(~2027 중반) 시 재채점 없이 평가. 2026 4-7월
  사후 채점은 shadow backfill이며, 진짜 전향은 2026-08-01 사전 채점부터.
- **재현 코드**: `results/phase3/scripts/` (전 실험, 게이트 포함).
- **기각 이력**: PROJECT_MEMORY.md 4절 — 재시도 방지용 원장.

## KBO로 가져갈 것 / 가져가지 않을 것

- **가져갈 것**: 전향 코호트 정의, feature 계산법(공개 수준 데이터로
  충분), 검열·재보정 절차, 역할-aware 평가와 quota 정책, blackout·새
  정보 분해 같은 검증 장치, 그리고 "조건부 backtest + 대역" 서술 규율.
- **가져가지 않을 것**: MLB 계수(재적합 필수), quota 값(재선택),
  트래킹 증분 서사, IL 이력의 조기경보 활용, 단일 수치 성능 주장.
- **기대 관리**: MLB novel-pitcher stress 참조 대역 ~0.56-0.68 —
  **KBO 성능 예측치가 아니다** (KBO 성능은 재적합 후에만 측정 가능).
  제안서에는 "예측"이 아니라 "낯선 투수군에서의 MLB 성능 저하 폭"
  이라는 참조로만 쓰는 것이 과대약속을 막는 안전장치다.
